Inside FeatureBench

A benchmark for end-to-end development of complex features

FeatureBench evaluates agentic coding beyond bug fixing. It targets feature-level implementation with execution-based verification and explicit callable interfaces, so scoring is reproducible and grounded in real repository behavior.

Collection Pipeline

Scalable, test-driven instance construction from repositories

The pipeline validates fail-to-pass and pass-to-pass tests, traces runtime dependencies, extracts feature patches, and then post-verifies the modified codebase. This enables automated generation of executable and continually updatable benchmark instances with minimal manual intervention.

  • Dependency graph is built from dynamic tracing, not static heuristics only.
  • Post verification enforces F2P fail and P2P pass conditions before patch replay.
  • Problem statements are synthesized with explicit interfaces and import paths.

Empirical Results

Frontier agents still face a large feature-development gap

In the paper baseline, Claude Opus 4.5 reaches 74.4% resolved on SWE-bench but only 11.0% on FeatureBench Full, while GPT-5.1-Codex reaches 12.5%. The gap suggests current agents are still far from stable real-world feature shipping.
